# KorPIX 시스템 아키텍처
 
## 전체 구성
 
```
┌──────────────────────────────────────────────────────────┐
│                   KorPIX Ecosystem                        │
│                                                          │
│  ┌─────────────┐   ┌─────────────┐   ┌───────────────┐  │
│  │    Trust    │   │   Policy    │   │    Audit      │  │
│  │  Terminal   │──▶│   Engine    │──▶│   Network     │  │
│  │  (단말)     │   │  (정책검증) │   │  (감사기록)   │  │
│  └─────────────┘   └─────────────┘   └───────────────┘  │
│         │                 │                              │
│         └─────────────────┼──────────────────────────── │
│                           ▼                              │
│                  ┌─────────────────┐                     │
│                  │ Execution       │                     │
│                  │ Gateway         │                     │
│                  │ (외부 시스템)   │                     │
│                  └─────────────────┘                     │
└──────────────────────────────────────────────────────────┘
```
 
## 서비스 포트
 
| 서비스 | 포트 | 역할 |
|---|---|---|
| Policy Engine | 8001 | 행동 평가 및 승인 결정 |
| Audit Network | 8002 | 감사 기록 수집 및 조회 |
| Execution Gateway | 8003 | 외부 API 실행 |
 
## 데이터 흐름
 
1. AI 에이전트가 Trust Terminal에서 행동 요청 생성
2. Policy Engine이 Risk Score 계산 후 결정 반환
3. 승인된 행동은 Execution Gateway를 통해 실행
4. 모든 행동은 Audit Network에 해시 체인으로 기록
